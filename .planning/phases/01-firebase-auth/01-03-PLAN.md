```javascript
   AUTH FUNCTIONS

const signInWithApple = async () => {
  const { FirebaseAuthentication } = Capacitor.Plugins;
  const result = await FirebaseAuthentication.signInWithApple({
    skipNativeAuth: true,
  });
  const provider = new firebase.auth.OAuthProvider('apple.com');
  const credential = provider.credential({
    idToken: result.credential?.idToken,
    rawNonce: result.credential?.nonce,
  });
  await firebase.auth().signInWithCredential(credential);
};

const signInWithGoogle = async () => {
  const { FirebaseAuthentication } = Capacitor.Plugins;
  const result = await FirebaseAuthentication.signInWithGoogle();
  const credential = firebase.auth.GoogleAuthProvider.credential(
    result.credential?.idToken
  );
  await firebase.auth().signInWithCredential(credential);
};

const writeUserProfile = async (user) => {
  await firestore.collection('users').doc(user.uid).set({
    uid: user.uid,
    displayName: user.displayName || '',
    email: user.email || '',
    createdAt: firebase.firestore.FieldValue.serverTimestamp(),
    tier: 'free',
    subscriptionExpiry: null
  }, { merge: true });
};

const signOutUser = async () => {
  const { FirebaseAuthentication } = Capacitor.Plugins;
  await FirebaseAuthentication.signOut();
  await firebase.auth().signOut();
};
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User device to Firebase Auth | OAuth credentials cross from native iOS layer to Firebase Auth backend |
| App to Firestore | User profile data written to Firestore users collection |
| Native plugin to WebView | Capacitor bridge passes OAuth tokens from native to JS layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Spoofing | signInWithCredential | mitigate | Firebase Auth validates OAuth tokens server-side; app never stores raw tokens; the plugin and Firebase SDK handle the full token lifecycle |
| T-01-07 | Tampering | Firestore users collection | mitigate | `merge: true` prevents overwriting existing tier data; Firestore security rules (Phase 2, CLOUD-04) will enforce users can only write their own document |
| T-01-08 | Information Disclosure | User email/name in ProfileSheet | accept | Displayed only to the authenticated user on their own device; standard for profile screens |
| T-01-09 | Denial of Service | onAuthStateChanged hang | mitigate | Guard with SplashScreen that hides after auth check completes; if onAuthStateChanged never fires after 5 seconds, treat as unauthenticated and show login |
| T-01-10 | Elevation of Privilege | tier field set client-side | mitigate | tier defaults to 'free'; merge: true prevents client-side overwrites once subscription system (Phase 5) manages this field server-side; Phase 2 Firestore rules will restrict tier field writes |
</threat_model>
