<objective>
Implement free tier usage limits: query Firestore for NDAs created in the current month, display usage count in the ACCOUNT tab, and block NDA creation when the free tier limit (2/month) is reached with an upgrade prompt.

Purpose: Usage limits are the core monetization gate. Without them, there's no reason to subscribe. The counter must be query-based (not a stored number) so it can't be manipulated and resets naturally each month.

Output: Free users see "1 of 2 NDAs this month" in ACCOUNT and get blocked with an upgrade prompt on the 3rd attempt. Pro users (tier: "pro") see no limit.
</objective>

**Step 1:** Add getMonthlyUsage function in the auth functions section (after writeUserProfile):

```javascript
const getMonthlyUsage = async (uid) => {
  const now = new Date();
  const firstOfMonth = new Date(now.getFullYear(), now.getMonth(), 1);
  const snapshot = await firestore.collection('users').doc(uid)
    .collection('ndas')
    .where('createdAt', '>=', firebase.firestore.Timestamp.fromDate(firstOfMonth))
    .get();
  return snapshot.size;
};
```

**Step 2:** Add getUserTier function:

```javascript
const getUserTier = async (uid) => {
  const doc = await firestore.collection('users').doc(uid).get();
  if (doc.exists) return doc.data().tier || 'free';
  return 'free';
};
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | Client-side limit bypass | mitigate | Usage count comes from Firestore query (server-side count). Client can't fake the count. Phase 5 adds server-side validation via Cloud Functions for subscription verification. |
| T-02-08 | Elevation of Privilege | Fake tier: "pro" in user doc | mitigate | Firestore rules (from Plan 02-01) block client writes to tier field. Only server can set tier. Until Phase 5, all users are free tier. |
| T-02-09 | Spoofing | Deleting NDA docs to reset count | accept | User could delete their own NDA docs to reset count. Acceptable: they'd lose their NDA records. Phase 5 can add server-side creation logging if needed. |
</threat_model>
