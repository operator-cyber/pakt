  <action>
Create a `firestore.rules` file in the project root with the following security rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only access their own document
    match /users/{userId} {
      allow read: if request.auth != null && request.auth.uid == userId;
      allow create: if request.auth != null && request.auth.uid == userId;
      allow update: if request.auth != null && request.auth.uid == userId
                    && (!request.resource.data.diff(resource.data).affectedKeys()
                        .hasAny(['tier', 'subscriptionExpiry']));
      // tier and subscriptionExpiry are server-managed (Phase 5)
      // Client can update displayName, email, etc. but not tier

      // NDA subcollection
      match /ndas/{ndaId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }

    // Deny everything else
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

Then create a `storage.rules` file:
```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{userId}/{allPaths=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    // Deny everything else
    match /{allPaths=**} {
      allow read, write: if false;
    }
  }
}
```

**Note:** These rules files are for documentation and Firebase CLI deployment. Since we're not using Firebase CLI in this project, the rules need to be manually pasted into the Firebase Console. Add instructions to the human verification step.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | tier field in user doc | mitigate | Firestore rules block client writes to tier and subscriptionExpiry fields. Only server (Phase 5) can modify these. |
| T-02-02 | Information Disclosure | Cross-user data access | mitigate | Rules enforce request.auth.uid == userId on all paths. No wildcard reads. |
| T-02-03 | Tampering | User modifying another user's NDA | mitigate | NDA subcollection inherits userId path check from parent rule. |
</threat_model>
